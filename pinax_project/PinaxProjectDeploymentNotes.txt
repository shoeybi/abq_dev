====================
building the project
====================

- update the operating system
 $> sudo apt-get update

- install setup tools
 $> sudo apt-get install python-setuptools

- install pip
 $> sudo easy_install pip

- install virtualenv
 $> sudo pip install virtualenv

- install Pinax (only if virtual environment is not used)
 $> sudo pip install Pinax

- create a virtual environment
 $> virtualenv abaqual-env

- activate virtual environment
 $> source ./abaqual-env/bin/activate

- under this envitonment, install Pinax
 $> pip install Pinax

- list the projects available
 $> pinax-admin setup_project -l

- change "pip.call_subprocess" to "pip.util.call_subprocess" in the file "~/abaqual-env/local/lib/python2.7/site-packages/pinax/core/management/commands/setup_project.py"

- create a project of type "account"
 $> pinax-admin setup_project -b account abaqual-website

- build the website database
 $> cd ~/abaqual-website
 $> python manage.py syncdb


==================================
Apache and mod_wsgi for deployment
==================================

- install apache2 and mod_wsgi
 $> sudo apt-get install apache2
 $> sudo apt-get install libapache2-mod-wsgi

- add the following configuration to "/etc/apache2/sites-available/default"
      	WSGIDaemonProcess abaqual-website python-path=/home/ubuntu/abaqual-env/lib/python2.7/site-packages
        WSGIProcessGroup abaqual-website

        WSGIScriptAlias / /home/ubuntu/abaqual-website/wsgi.py
        <Directory /home/ubuntu/abaqual-website>
                Order deny,allow
                Allow from all
        </Directory>

- in the project settings ("settings.py"), include the full path in the database name. The database should look like:
     DATABASES = {
     	       "default": {
               		  "ENGINE": "django.db.backends.sqlite3", 
			  "NAME": "/home/ubuntu/abaqual-website/dev.db", 
        		  "USER": "",                             
        		  "PASSWORD": "",                         
        		  "HOST": "",                             
        		  "PORT": "",                             
    			  }
		}

- restart the apache web-service
 $> sudo service apache2 restart
